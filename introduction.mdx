RideShare Live is a production-ready ride-sharing platform designed for self-hosting. It connects riders and drivers through a real-time, event-driven architecture built on Spring Boot 3 and React 19. Riders book trips, drivers accept requests, and both sides track the ride in real time over a STOMP WebSocket connection.

## Key features

<Tabs>
  <Tab title="Auth & security">
    - JWT access tokens signed with HMAC-SHA256, issued on login
    - OTP verification via Gmail SMTP — new accounts must verify email before signing in
    - Refresh token rotation stored in the `refresh_tokens` table
    - Role-based access control: `USER` (rider), `DRIVER`, and `ADMIN`
    - BCrypt password hashing via Spring Security
    - Stateless session management — no server-side sessions
  </Tab>
  <Tab title="Real-time tracking">
    - STOMP over WebSocket with two endpoints: `/ws` (native) and `/ws-sockjs` (SockJS fallback)
    - In-bound channel interceptor validates the JWT on every STOMP `CONNECT` frame
    - Simple in-memory broker on `/topic` and `/queue` destinations
    - Driver location updates broadcast to riders over topic subscriptions
    - In-app notification center for ride and payment events
  </Tab>
  <Tab title="Payments">
    - Stripe Checkout session creation via the `stripe-java` SDK (v26.12.0)
    - Session verification on return to confirm payment before marking a ride paid
    - Transaction history persisted to the `payments` table
    - Refund management through the Stripe dashboard
  </Tab>
  <Tab title="Dashboards">
    - **Rider dashboard** — book rides, estimate fare, view ride history, track active trip in real time
    - **Driver dashboard** — see incoming ride requests, accept or reject, update ride milestones, monitor earnings
    - Quick fare estimator on the landing page calls `GET /rides/estimate` without requiring a login
    - Live map panel using browser Geolocation and a tile-based map renderer
  </Tab>
</Tabs>

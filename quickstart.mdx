---
title: "Quickstart"
description: "Get RideShare Live running locally — backend API on port 8080 and React frontend on port 5173."
---

This guide walks you through cloning the repository, configuring the environment, and starting both the Spring Boot backend and the Vite frontend from scratch.

<Steps>
  <Step title="Install prerequisites">
    You need the following tools installed before continuing.

    | Tool | Minimum version | Check |
    |---|---|---|
    | Java | 17 | `java -version` |
    | Maven | 3.8 | `mvn -version` |
    | PostgreSQL | 14 | `psql --version` |
    | Redis | 6 | `redis-cli --version` |
    | Node.js | 18 | `node --version` |
    | Git | any | `git --version` |

    <CodeGroup>

    ```bash macOS
    # Java 17
    brew install openjdk@17
    echo 'export PATH="/opt/homebrew/opt/openjdk@17/bin:$PATH"' >> ~/.zshrc
    source ~/.zshrc

    # Maven
    brew install maven

    # PostgreSQL
    brew install postgresql@14
    brew services start postgresql@14

    # Redis
    brew install redis
    brew services start redis

    # Node.js 18+
    brew install node@18
    ```

    ```bash Linux (Ubuntu/Debian)
    # Java 17
    sudo apt-get update
    sudo apt-get install -y openjdk-17-jdk

    # Maven
    sudo apt-get install -y maven

    # PostgreSQL
    sudo apt-get install -y postgresql postgresql-contrib
    sudo systemctl enable --now postgresql

    # Redis
    sudo apt-get install -y redis-server
    sudo systemctl enable --now redis-server

    # Node.js 18+
    curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
    sudo apt-get install -y nodejs
    ```

    ```powershell Windows
    # Java 17
    winget install Oracle.JDK.17

    # Maven — download from https://maven.apache.org/download.cgi
    # and add bin/ to PATH

    # PostgreSQL
    winget install PostgreSQL.PostgreSQL

    # Redis (via Docker)
    docker run -d -p 6379:6379 --name redis redis:latest

    # Node.js 18+
    winget install OpenJS.NodeJS.LTS
    ```

    </CodeGroup>
  </Step>

  <Step title="Clone the repository">
    Clone the monorepo. It contains two top-level directories: `ridesharelive-backend-main/` and `ridesharelive-frontend-main/`.

    ```bash
    git clone https://github.com/sumitdhondikar26-lgtm/Ride-Share
    cd Ride-Share
    ```
  </Step>

  <Step title="Configure the backend">
    Copy the example environment file and fill in your values.

    ```bash
    cp ridesharelive-backend-main/.env.example ridesharelive-backend-main/.env
    ```

    Open `.env` and set the following variables:

    ```bash ridesharelive-backend-main/.env
    # Database
    DB_HOST=localhost
    DB_PORT=5432
    DB_NAME=ridesharelive
    DB_USERNAME=ridesharelive_user
    DB_PASSWORD=secure_password

    # JWT
    JWT_SECRET=your_super_secret_jwt_key_minimum_256_bits_long
    JWT_EXPIRATION=86400000

    # Email (Gmail SMTP)
    MAIL_HOST=smtp.gmail.com
    MAIL_PORT=587
    MAIL_USERNAME=yourgmail@gmail.com
    MAIL_PASSWORD=your_16_char_google_app_password
    MAIL_FROM_ADDRESS=yourgmail@gmail.com
    MAIL_FROM_NAME=RideShare Live

    # Stripe (optional for local dev without payments)
    STRIPE_API_KEY=your_stripe_api_key
    STRIPE_WEBHOOK_SECRET=your_webhook_secret
    ```

    <Warning>
      `JWT_SECRET` must be at least 256 bits (32 characters) long. A short secret causes the application to fail at startup with an `IllegalArgumentException` from the JJWT library.
    </Warning>

    To generate a Gmail App Password:
    1. Go to [https://myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords)
    2. Select **Mail** and your device
    3. Copy the generated 16-character password into `MAIL_PASSWORD`
  </Step>

  <Step title="Set up PostgreSQL">
    Create the database and a dedicated user. Run these commands as the `postgres` superuser.

    ```sql
    -- Connect as the postgres superuser
    -- psql -U postgres

    CREATE DATABASE ridesharelive;

    CREATE USER ridesharelive_user WITH PASSWORD 'secure_password';

    ALTER ROLE ridesharelive_user SET client_encoding TO 'utf8';

    GRANT ALL PRIVILEGES ON DATABASE ridesharelive TO ridesharelive_user;
    ```

    Verify the connection:

    ```bash
    psql -U ridesharelive_user -d ridesharelive -h localhost
    ```

    Spring Boot will create the schema on first run via Hibernate DDL auto-update. The `application.properties` maps to:

    ```properties
    spring.datasource.url=jdbc:postgresql://${DB_HOST}:${DB_PORT}/${DB_NAME}
    spring.datasource.username=${DB_USERNAME}
    spring.datasource.password=${DB_PASSWORD}
    spring.jpa.hibernate.ddl-auto=validate
    ```
  </Step>

  <Step title="Start the backend">
    Run the Spring Boot application from the backend directory.

    ```bash
    cd ridesharelive-backend-main
    mvn spring-boot:run
    ```

    Maven downloads dependencies on the first run (this may take a few minutes). Once you see the banner and `Started BackendApplication`, the API is listening on port 8080.

    You can also build and run the JAR directly:

    ```bash
    mvn clean package
    java -jar target/spring-boot-docker.jar
    ```

    <Note>
      On first run, the application connects to PostgreSQL and creates all tables. If you see a schema validation error, switch `spring.jpa.hibernate.ddl-auto` to `update` in `src/main/resources/application.properties` for the initial run, then revert to `validate`.
    </Note>
  </Step>

  <Step title="Install and start the frontend">
    Open a second terminal and start the React development server.

    ```bash
    cd ridesharelive-frontend-main
    npm install
    npm run dev
    ```

    Vite compiles the app and opens it on port 5173. The dev server proxies API calls to `http://localhost:8080` automatically.

    To run the test suite:

    ```bash
    # Unit tests
    npm test

    # End-to-end tests
    npm run test:e2e
    ```
  </Step>

  <Step title="Access the app">
    With both servers running, open your browser:

    | URL | Description |
    |---|---|
    | `http://localhost:5173` | React frontend — Login / Signup |
    | `http://localhost:8080/swagger-ui.html` | Interactive API documentation |
    | `http://localhost:8080/v3/api-docs` | Raw OpenAPI JSON |
    | `http://localhost:8080/health` | Health check endpoint |

    Create an account by clicking **Sign up** on the landing page. Choose the **Rider** or **Driver** role. After submitting the form, you will be redirected to the appropriate dashboard.

    <Note>
      On first signup, the backend sends a 6-digit OTP to your registered email address via Gmail SMTP. You must enter this OTP to verify your account before you can log in. Make sure `MAIL_USERNAME` and `MAIL_PASSWORD` are correctly set in `.env`, and that the Gmail account has Less Secure App Access or an App Password enabled.
    </Note>
  </Step>
</Steps>

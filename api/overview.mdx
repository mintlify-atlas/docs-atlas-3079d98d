## Base URL

```
http://localhost:8080
```

In production, set `APP_URL` to your deployed domain (e.g. `https://api.yourdomain.com`). All endpoints are served under the `/api/v1` context path, as configured by `server.servlet.context-path=/api/v1` in `application.properties`.

Full example:

```
POST http://localhost:8080/api/v1/auth/login
```

## Authentication

Most endpoints require a JWT access token. Include it as a Bearer token in the `Authorization` header:

```bash
curl -X GET http://localhost:8080/api/v1/rides/history \
  -H "Authorization: Bearer <your-access-token>"
```

Obtain a token by calling [POST /auth/login](/api/auth). The response includes `token`, `refreshToken`, and `expiresIn` (seconds).

## Request / Response Format

All request and response bodies use JSON. Set `Content-Type: application/json` on requests that include a body.

```bash
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email": "rider@example.com", "password": "secret"}'
```

## Error Codes

| Code | Meaning |
|---|---|
| `200` | Success |
| `201` | Created |
| `400` | Bad request — missing or invalid fields |
| `401` | Unauthorized — missing, expired, or invalid token |
| `403` | Forbidden — authenticated but not permitted |
| `404` | Resource not found |
| `429` | Rate limited — too many requests (e.g. OTP requests) |
| `502` | Bad gateway — upstream service error (e.g. Stripe) |
| `503` | Service unavailable — upstream not configured (e.g. Stripe key missing) |

Error responses return a JSON body with a `message` field:

```json
{
  "message": "email and password are required."
}
```

Rate-limit responses also include `retryAfterSeconds`:

```json
{
  "message": "Too many OTP requests. Please wait before trying again.",
  "retryAfterSeconds": 60
}
```

## Health Check

Two public endpoints are available without authentication:

```bash
# Liveness check
curl http://localhost:8080/health
# {"status":"UP"}

# Root
curl http://localhost:8080/
# {"message":"Backend is running"}
```

<Note>
  The health endpoints are at the root path — **not** under `/api/v1`. They are served directly by the Spring Boot application before the context path is applied.
</Note>

RideShare Live uses Spring WebSocket with STOMP messaging. Clients subscribe to topics to receive push updates without polling.

<Warning>
  A valid JWT access token is required to authenticate a WebSocket connection. Connections made without a token (or with an expired token) are accepted by the server but are treated as unauthenticated — user-specific destinations (`/user/queue/...`) will not deliver messages.
</Warning>

## Endpoints

| Endpoint | Transport | Notes |
|---|---|---|
| `ws://localhost:8080/ws` | Native WebSocket | Use with `@stomp/stompjs` directly |
| `http://localhost:8080/ws-sockjs` | SockJS (HTTP fallback) | Use with `sockjs-client` + `@stomp/stompjs` |

`SPRING_PROFILES_ACTIVE` and `APP_URL` affect the host; update the URL accordingly in production.

## Message Broker

The server configures a simple in-memory STOMP broker:

| Prefix | Purpose |
|---|---|
| `/topic` | Broadcast destinations (all subscribers receive the message) |
| `/queue` | Point-to-point destinations |
| `/user` | User-specific destination prefix (routed to individual sessions) |
| `/app` | Application destination prefix — messages sent here are routed to `@MessageMapping` handlers |

## Authentication

Pass the JWT access token in the STOMP `CONNECT` frame. The `WebSocketAuthChannelInterceptor` checks **two header names** on the CONNECT frame:

1. `Authorization: Bearer <token>` — standard Bearer format
2. `token: <token>` — bare token value (no `Bearer` prefix)

If the token is valid, the WebSocket session is associated with the authenticated user and user-specific queue subscriptions work correctly.

<Note>
  The JWT is inspected once at CONNECT time. If the access token expires during a long-lived session, the client must reconnect with a fresh token. Use `/auth/refresh` to obtain a new access token before reconnecting.
</Note>

## Connecting with @stomp/stompjs

<CodeGroup>
```javascript Native WebSocket
import { Client } from '@stomp/stompjs';

const token = localStorage.getItem('accessToken');

const client = new Client({
  brokerURL: 'ws://localhost:8080/ws',

  // Pass JWT in the CONNECT frame headers
  connectHeaders: {
    Authorization: `Bearer ${token}`,
  },

  // Optional: tune heartbeat intervals (milliseconds)
  heartbeatIncoming: 4000,
  heartbeatOutgoing: 4000,

  onConnect: () => {
    console.log('Connected to RideShare Live WebSocket');
  },

  onDisconnect: () => {
    console.log('Disconnected');
  },

  onStompError: (frame) => {
    console.error('STOMP error', frame);
  },
});

client.activate();
```

```javascript SockJS fallback
import { Client } from '@stomp/stompjs';
import SockJS from 'sockjs-client';

const token = localStorage.getItem('accessToken');

const client = new Client({
  webSocketFactory: () => new SockJS('http://localhost:8080/ws-sockjs'),

  connectHeaders: {
    Authorization: `Bearer ${token}`,
  },

  heartbeatIncoming: 4000,
  heartbeatOutgoing: 4000,

  onConnect: () => {
    console.log('Connected via SockJS');
  },
});

client.activate();
```
</CodeGroup>

## Subscribing to Ride Updates

Once connected, subscribe to the topics relevant to your use case:

<CodeGroup>
```javascript Rider — subscribe to own ride updates
client.onConnect = () => {
  // Receive updates for a specific ride (broadcast to all subscribers of this topic)
  client.subscribe(`/topic/ride/${rideId}`, (message) => {
    const ride = JSON.parse(message.body);
    console.log('Ride update:', ride);
  });

  // Receive personal notifications delivered to this user's queue
  client.subscribe('/user/queue/notifications', (message) => {
    const notification = JSON.parse(message.body);
    console.log('New notification:', notification);
  });
};
```

```javascript Driver — subscribe to requested rides
client.onConnect = () => {
  // All new ride requests are broadcast here
  client.subscribe('/topic/rides/requested', (message) => {
    const rides = JSON.parse(message.body);
    console.log('New ride requests:', rides);
  });

  // Driver-specific queue for accepted/cancelled events
  client.subscribe('/user/queue/ride-updates', (message) => {
    const update = JSON.parse(message.body);
    console.log('Ride update for driver:', update);
  });
};
```
</CodeGroup>

## Full Connection Example

```javascript
import { Client } from '@stomp/stompjs';

function connectWebSocket(token, rideId, onRideUpdate, onNotification) {
  const client = new Client({
    brokerURL: 'ws://localhost:8080/ws',
    connectHeaders: { Authorization: `Bearer ${token}` },
    heartbeatIncoming: 4000,
    heartbeatOutgoing: 4000,
    reconnectDelay: 5000, // auto-reconnect after 5 s
  });

  client.onConnect = () => {
    client.subscribe(`/topic/ride/${rideId}`, (msg) => {
      onRideUpdate(JSON.parse(msg.body));
    });

    client.subscribe('/user/queue/notifications', (msg) => {
      onNotification(JSON.parse(msg.body));
    });
  };

  client.activate();
  return client; // call client.deactivate() to close the connection
}
```

## Disconnecting

```javascript
// Gracefully close the STOMP session and underlying WebSocket
client.deactivate();
```
